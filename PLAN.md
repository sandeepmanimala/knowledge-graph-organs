# 🧬 Plan for Expanding and Optimizing the Multi-Omics Knowledge Graph

This plan outlines the strategic roadmap for making the `knowledge-graph-organs` more powerful and useful. Based on the technical requirements for scale and computational limits, we have chosen to focus purely on the integration of the **Metabolomics Dimension** and high-performance ingestion.

## Phase 1: Expanding Data & Dimensions

To elevate the graph from its current 7 dimensions to a true comprehensive in-silico human model, we will integrate a high-impact, systemic dataset:

### 1.1 Metabolomics Integration (HMDB)
*   **Data Source:** Human Metabolome Database (HMDB). A massive ~5GB XML file detailing over 300k metabolites.
*   **Integration Point:** Adding `Metabolite` nodes. Connecting them via `BINDS_TO` to `Protein` nodes (Uniprot IDs) and `ASSOCIATED_WITH` to `Disease` nodes (DisGeNET).
*   **Implementation Details:**
    *   **Acquisition (`scripts/1_acquire/hmdb/download_hmdb.py`):** A custom streaming HTTP downloader to handle gigabyte-scale zip extraction without OOM issues.
    *   **Parsing (`scripts/2_clean/hmdb/parse_hmdb.py`):** Uses Python's `xml.etree.ElementTree.iterparse` to parse the 5GB XML node-by-node incrementally. It explicitly outputs CSV files matching the strict typing requirements of the `neo4j-admin import` tool (e.g., `:ID`, `:LABEL`, `:START_ID`, `:END_ID`, `:TYPE`).

## Phase 2: Performance & Scale Optimizations

Given the massive influx of edges generated by connecting 300,000 metabolites to millions of existing proteins and diseases across the 54 organs, the ingestion layer must be optimized.

### 2.1 Refactoring Ingestion (`scripts/3_graph/import_hmdb_admin.sh`)
The current Cypher `UNWIND` method is excellent for incremental updates but catastrophic for initial bulk ingestion of tens of millions of rows.

*   **Action:** Transition from transactional Cypher ingestion to the **Neo4j Admin Import tool**.
*   **Implementation:** The script uses `sudo -u neo4j neo4j-admin database import full` to completely bypass the database's transactional log, parsing CSV files directly into the Neo4j binary store (`neo4j/data/databases`). This drops ingestion time for the Metabolomics dimension from *hours* down to *minutes*.
*   **Indexing:** Post-import, the script automatically deploys Cypher indexes (`CREATE INDEX`) on `metaboliteId` and `name` to ensure sub-second retrieval times when visualizing pathways on the Streamlit dashboard.
